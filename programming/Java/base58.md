# Base58
> 기존 base64의 경우, 혼동되는 문자들(0, O, I, l) 그리고 query를 방해하는 문자(=,+...)가 있다는 단점을 보완하기 위해 나온 인코딩 기법 📝

+ 처리방식
   + base64
      + 64진수 체계의 인코딩 기법
      + 입력된 값을 6bit씩 구분하여, 지정된 table의 charater로 치환   
      + 
   + **base58**
      + 입력된 값을 big number로 변경하여 58로 나눠, 지정된 table의 character로 치환
